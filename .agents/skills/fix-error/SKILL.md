# 구조화된 에러 진단 및 해결 절차 (Vesper)

이 스킬은 React Native 또는 Supabase Edge Functions 개발 중 에러 발생 시 사용됩니다.

1. **에러 원문 수집:** 콘솔이나 로그에 나타난 에러 메시지와 스택 트레이스를 확인합니다.
2. **현상 분석:** 무엇을 하려고 했을 때 에러가 발생했는지 정리합니다.
3. **가설 설정:** 에러의 원인이 될 수 있는 가설 3가지를 도출합니다. (예: 타입 불일치, 비동기 타임아웃, DB 쿼리 권한)
4. **코드 검증:** 관련된 코드를 검색 도구를 사용해 확인합니다.
5. **테스트 및 수정 제안:** 가장 가능성 높은 원인을 수정하는 코드 패치를 제안합니다.
6. **Fallback 여부 확인:** 외부 API(LLM/Search) 문제인 경우 Fallback 응답이 제대로 작동했는지 검증합니다.
7. **최종 보고:** 수정 내역을 요약하여 사용자에게 보고합니다.
